# v2.1 Public Key Certificate

This flow produces a PKC from a v2.x manifest.

## Inputs

Download these input files into your working directory, or substitute your own local paths:

- Issuer certificate: [TestCA.mldsa65.cert.example.pem][res-testca-mldsa-cert]
- Issuer private key: [TestCA.mldsa65.private.example.pem][res-testca-mldsa-key]
- Holder certificate: [TestCA.cert.example.pem][res-testca-cert]
- Attributes JSON: [localhost-policyreference-v2.json][res-different-policy-v2]
- Components JSON: [localhost-componentlistv3adv.json][res-different-components-v3-adv]
- Extensions JSON: [extentionspkc.json][res-different-extensions-pkc]

If you do not want to use the linked test issuer material, use [Generate Local Demo PKI](../../getting-started.md#generate-local-demo-pki) and replace the issuer certificate and local-key filenames below.

## Verify the target Platform Certificate version

1. Reference the Attribute JSON file referenced in the [Inputs](<#inputs>) section.
2. Follow [these instructions](<../change-target-version.md#tcg-certificate-specification-definition>).
3. Verify it reflects majorVersion 2 and minorVersion 1.

## Generate the envelope

```bash
bin/paccor certgen \
  --serial 1891 \
  --not-before 20240101 \
  --not-after 20300101 \
  --issuer-cert TestCA.mldsa65.cert.example.pem \
  --holder-cert TestCA.cert.example.pem \
  --attributes-json localhost-policyreference-v2.json \
  --components-json localhost-componentlistv3adv.json \
  --extensions-json extentionspkc.json \
  --sig-profile ml-dsa-65 \
  --finalize \
  --out v21-pkc-envelope.json
```

## Assemble the envelope

Choose one signing mode:

=== "Local key"

    ```bash
    bin/paccor assemble \
      --in v21-pkc-envelope.json \
      --out v21-pkc-cert.pem \
      --pem \
      --local-key TestCA.mldsa65.private.example.pem \
      --issuer-cert TestCA.mldsa65.cert.example.pem
    ```

=== "PKCS#11"

    ```bash
    bin/paccor assemble \
      --in v21-pkc-envelope.json \
      --out v21-pkc-cert.pem \
      --pem \
      --issuer-cert TestCA.mldsa65.cert.example.pem \
      --pkcs11-module /path/to/pkcs11.so \
      --pkcs11-slot 0 \
      --pkcs11-key-alias signing-key \
      --pkcs11-pin-file pkcs11.pin
    ```

=== "Remote signer"

    ```bash
    bin/paccor assemble \
      --in v21-pkc-envelope.json \
      --out v21-pkc-cert.pem \
      --pem \
      --issuer-cert TestCA.mldsa65.cert.example.pem \
      --remote-url https://signer.example/sign \
      --remote-auth bearer:REPLACE_ME
    ```

=== "Detached signature"

    ```bash
    bin/paccor assemble \
      --in v21-pkc-envelope.json \
      --out v21-pkc-cert.pem \
      --pem \
      --issuer-cert TestCA.mldsa65.cert.example.pem \
      --signature "$(cat v21-pkc.sig.b64)" \
      --sig-encoding der
    ```

For the full option set, see [Signing Modes](../signing-options.md).

## Validate

```bash
bin/paccor validate \
  --pkcPlatformCert v21-pkc-cert.pem \
  --publicKeyCert TestCA.mldsa65.cert.example.pem \
  --components-json localhost-componentlistv3adv.json
```
