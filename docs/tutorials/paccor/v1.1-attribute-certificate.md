# v1.1 Attribute Certificate

This is the direct compatibility path for the v1.1 platform certificate.

## Inputs

Download these input files into your working directory, or substitute your own local paths:

- Issuer certificate: [TestCA.cert.example.pem][res-testca-cert]
- Issuer private key: [TestCA.private.example.pem][res-testca-key]
- Holder certificate: [TCG_EK_ecc_p384_P-384_Test.pem][res-holder-ek]
- Attributes JSON: [base-bare-bones-policyreference.json][res-bare-policy]
- Components JSON: [base-bare-bones-componentlist.json][res-bare-components]
- Extensions JSON: [base-bare-bones-extentions-no-ti.json][res-bare-extensions-no-ti]

If you do not want to use the linked test issuer material, use [Generate Local Demo PKI](../../getting-started.md#generate-local-demo-pki) and replace the issuer certificate and local-key filenames below.

## Generate the envelope

```bash
bin/paccor certgen \
  --serial 1 \
  --not-before 20240101 \
  --not-after 20300101 \
  --issuer-cert TestCA.cert.example.pem \
  --holder-cert TCG_EK_ecc_p384_P-384_Test.pem \
  --attributes-json base-bare-bones-policyreference.json \
  --components-json base-bare-bones-componentlist.json \
  --extensions-json base-bare-bones-extentions-no-ti.json \
  --sig-profile rsa-sha256 \
  --finalize \
  --out v11-ac-envelope.json
```

## Assemble the envelope

Choose one signing mode:

=== "Local key"

    ```bash
    bin/paccor assemble \
      --in v11-ac-envelope.json \
      --out v11-ac-cert.pem \
      --pem \
      --local-key TestCA.private.example.pem \
      --issuer-cert TestCA.cert.example.pem
    ```

=== "PKCS#11"

    ```bash
    bin/paccor assemble \
      --in v11-ac-envelope.json \
      --out v11-ac-cert.pem \
      --pem \
      --issuer-cert TestCA.cert.example.pem \
      --pkcs11-module /path/to/pkcs11.so \
      --pkcs11-slot 0 \
      --pkcs11-key-alias signing-key \
      --pkcs11-pin-file pkcs11.pin
    ```

=== "Remote signer"

    ```bash
    bin/paccor assemble \
      --in v11-ac-envelope.json \
      --out v11-ac-cert.pem \
      --pem \
      --issuer-cert TestCA.cert.example.pem \
      --remote-url https://signer.example/sign \
      --remote-auth bearer:REPLACE_ME
    ```

=== "Detached signature"

    ```bash
    bin/paccor assemble \
      --in v11-ac-envelope.json \
      --out v11-ac-cert.pem \
      --pem \
      --issuer-cert TestCA.cert.example.pem \
      --signature "$(cat v11-ac.sig.b64)" \
      --sig-encoding der
    ```

For the full option set, see [Signing Modes](../signing-options.md).

## Validate

```bash
bin/paccor validate \
  --x509v2AttrCert v11-ac-cert.pem \
  --publicKeyCert TestCA.cert.example.pem \
  --components-json base-bare-bones-componentlist.json
```

## Notes

v1.1 is AC-only. If you need a PKC, use the [v2.1 Public Key Certificate](v2.1-public-key-certificate.md) tutorial.
